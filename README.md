# OCHA AI Prototypes

This repository hosts **exploratory, AI‑assisted prototypes and visualisations** created by OCHA staff and partners for learning, experimentation, and internal discussion.

⚠️ **All content here is experimental.**  
It is **not validated, not authoritative, and not for operational or
decision‑making use**.

---

## Purpose

The goal of this repository is to enable **safe, visible innovation** by providing a controlled environment where colleagues can:

- Experiment with AI‑assisted “vibe coding”
- Prototype dashboards, visuals, and small web apps
- Explore ideas and interactions
- Share early concepts with colleagues for feedback

This repository is **not** intended to host production systems or operational tools.

---

## Governance and classification

This repository is an **approved technical environment for Tier 1 – Shareable Prototypes** under OCHA’s *AI Vibe Coding Guidance*.

All artefacts hosted here must:
- Be clearly labelled as **prototype / exploratory**
- Use **non‑sensitive data only**
- Have a **named human owner**
- Declare **data sources**, **AI involvement**, and **known limitations**

If a prototype is used in briefings, planning, or decision‑making, it likely falls under **Tier 2** and should not remain here without appropriate review.

See:
- `GOVERNANCE.md` for detailed rules
- `CONTRIBUTING.md` for how to add a prototype

---

## What belongs here

Examples of appropriate content include:
- Prototype dashboards
- Interactive data visualisations
- Exploratory HTML / JavaScript web apps
- Demonstrations created using AI‑assisted coding tools

These outputs are intended for **learning, exploration, and discussion only**.

---

## What does NOT belong here

Do **not** use this repository for:
- Operational or decision‑support tools
- Production or public‑facing systems
- Authoritative datasets or figures
- Artefacts using sensitive, personal, or restricted data
- Anything intended to directly influence decisions
