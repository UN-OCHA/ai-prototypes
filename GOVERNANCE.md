# Governance for AI Prototypes

This repository is an **approved environment for Tier 1 – Shareable Prototypes** under OCHA’s *AI Vibe Coding Guidance*.

All artefacts hosted here are exploratory, non‑authoritative, and intended for learning, demonstration, and internal discussion only.

---

## Tiering and intent

Content in this repository is assumed to be:

**Tier 1 – Shareable prototype**
- Exploratory
- Internally shared
- Not validated
- Not authoritative
- Not operational

If a prototype:
- Is used in briefings
- Influences planning or decisions
- Is relied upon beyond exploration

➡️ It must be treated as **Tier 2** and should **not** remain here without appropriate review and escalation.

---

## Mandatory transparency (non‑negotiable)

Every prototype **must clearly display or document**, either in the UI or in a
README:

- **Human owner** (name and role)
- **Purpose** (what it is and is *not* intended to do)
- **Data sources** (including whether data is public, synthetic, or sample)
- **AI involvement** (what was generated or assisted by AI)
- **Known limitations** (incompleteness, assumptions, risks)

If this information is not visible, the artefact is **not ready to be shared**.

---

## Data and security boundaries

Across all content in this repository:

- Use **non‑sensitive data only**
- Do **not** include personal, confidential, or operationally sensitive data
- Do **not** commit secrets, tokens, API keys, or credentials
- Standard OCHA and UN data responsibility and information security rules apply

---

## Accountability

Use of AI does **not** remove human accountability.

Each prototype must have a **named owner** responsible for:
- Correct framing and caveats
- Appropriate data use
- Safe sharing
- Addressing issues raised through review

---

## Oversight and lifecycle

Maintainers may:
- Request changes or added disclaimers
- Pause or remove prototypes that drift toward Tier 2 use
- Recommend graduation to a different repository or hosting environment

Governance exists to **support safe innovation**, not to block experimentation,
while preventing false certainty and unmanaged risk.
