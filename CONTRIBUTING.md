# Contributing to OCHA AI Prototypes

Thank you for your interest in contributing.

This repository exists to support **safe experimentation and learning** through AI‑assisted prototypes and visualisations. Contributions are welcome — as long as they follow the governance and transparency rules described below.

---

## What this repository is for

This repository hosts **Tier 1 – Shareable Prototypes** under OCHA’s *AI Vibe Coding Guidance*.

Appropriate contributions include:
- Prototype dashboards
- Interactive visualisations
- Exploratory HTML / JavaScript web apps
- Demonstrations created using AI‑assisted “vibe coding”

All content here must be clearly labelled as **prototype / exploratory** and must **not** be used for operational or decision‑making purposes.

---

## What does NOT belong here

Please do **not** contribute:
- Operational or decision‑support tools
- Production or public‑facing systems
- Validated or authoritative figures
- Artefacts using sensitive, personal, or restricted data
- Anything intended to directly influence decisions

If your work starts to move in this direction, it likely falls under **Tier 2** and should be discussed before being shared here.

---
